ONE SHOT GAME PROTOCOL A user may ask an AI to play a "One Shot Game". Process: 1. The player describes a game idea. 2. The AI generates the game. 3. The output must be a complete single-file HTML game. Constraints: - single HTML document - embedded JavaScript and CSS - no external libraries - playable immediately in a browser Rule: The generation happens exactly once. No retries or regeneration.
