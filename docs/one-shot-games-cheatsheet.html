📋 The Three Rules What makes a game "one shot"
01
ONE FILE
All HTML, CSS, and JavaScript are inline in a single .html file. No build step. Open it in any browser — offline, emailed, hosted — it just works.
02
NO SAVES
When the tab closes, the game ends. No localStorage persistence between sessions. This is a design feature — every session is its own story. Live with your choices.
03
SELF-CONTAINED
No auth, no backend API calls, no databases. Google Fonts CDN is fine. The game must run with WiFi off. If it needs a server, it's not a one-shot game.

  No external deps except Google Fonts.

Visual style: [colors, fonts, vibe — e.g. CRT terminal, dark bg, neon amber]

🐛 Debugging Reference Common issues and fixes
Symptom Fix
Fonts look wrong Check <head> for Google Fonts <link> tag — Claude sometimes forgets it on long outputs
Works locally, breaks on GitHub Pages All hrefs must be relative filenames only — no leading /, no full URLs
Game too easy / too hard Tell Claude: "Increase difficulty by 30% — [specific mechanic to change]"
Doesn't work on mobile Add to your prompt: "Include touch controls alongside keyboard controls"
Claude output is garbled / looping Start a fresh conversation — paste clean requirements and game concept again
File is huge / slow to load Ask Claude to remove base64 assets and use CSS shapes or SVG instead
Collision detection is wrong Paste the specific JS function and ask Claude to debug it alone, not the whole file

🎨 Design Tips Make your arcade look like a real product
Define a style brief once. Paste it into every game prompt. Consistency is free and makes the arcade feel cohesive.
Pick 2–3 accent colors. Commit. Amber + cyan + green reads "terminal arcade." Stick to it across every file.
Name your arcade. "My Arcade" is temporary. A real name makes it a portfolio piece. Own it.
Write your own card descriptions. Claude's defaults are generic. One sentence of real voice goes a long way.
CRT scanline trick. Fixed pseudo-element with repeating-linear-gradient at low opacity. Free atmosphere, every page.
Lean into the constraint. Tell Claude "design for one-shot — the no-save rule should feel intentional, not like a missing feature."
