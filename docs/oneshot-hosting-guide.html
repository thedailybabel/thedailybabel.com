Step-by-Step Instructions
STEP 01 Create a GitHub Account

If you already have a GitHub account, skip ahead to Step 2. Otherwise:

  1. Go to https://github.com and click Sign up.
  2. Enter your email address, create a password, and choose a username. Your username will become part of your game's public URL, so pick something you're happy with.
  3. Verify your email address when GitHub sends you a confirmation.
  4. Log in. You're now a GitHub citizen.
ℹ
Free Tier Is Enough GitHub Pages is available on free accounts. You do not need GitHub Pro, Teams, or any paid plan to host static files.
STEP 02 Create a New Repository

A repository (repo) is the folder that holds all your game files on GitHub.

  1. Click the + icon in the top-right corner of GitHub and choose New repository.
  2. Give the repo a name. For a dedicated games site, name it exactly:
    YOUR-USERNAME.github.io
    Replacing YOUR-USERNAME with your actual GitHub username. This special naming convention activates the root GitHub Pages domain for your account.
  3. Set visibility to Public. GitHub Pages requires public repos on free accounts.
  4. Check Add a README file — this gives the repo something to start with so it's not empty.
  5. Click Create repository.
✦
Naming Options The username.github.io pattern gives you a root URL like https://username.github.io. If you'd prefer a sub-path (e.g. /games), name the repo anything else and your URL becomes https://username.github.io/repo-name. Either approach works fine for hosting games.
STEP 03 Enable GitHub Pages

GitHub Pages is the feature that turns your repo into a live website. It doesn't turn itself on automatically.

  1. Inside your new repo, click the Settings tab (top of the repo, right side).
  2. In the left sidebar, click Pages (under the "Code and automation" section).
  3. Under Source, select Deploy from a branch.
  4. Set the branch to main and the folder to / (root).
  5. Click Save.

After a minute or two, GitHub will display the live URL of your site at the top of the Pages settings panel:

LIVE URLhttps://YOUR-USERNAME.github.io
⚠
Propagation Delay The first deployment takes 1–3 minutes. Subsequent pushes are usually faster but can still take up to a minute to appear live. Don't panic if the URL 404s immediately.
STEP 04 Submit Your Prompt and Capture the Game
⚠
The One-Shot Rule One prompt. One LLM run. One HTML output. No follow-up prompts, no refinement, no do-overs. You take what the model gives you and ship it. Craft your prompt carefully before you pull the lever — the prompt is the skill, the game is the prize.

Go to your LLM of choice — Claude, ChatGPT, Gemini, or any other — and submit your prompt. Here is a proven example prompt for a first game:

EXAMPLE PROMPTCreate a complete single-file HTML5 Snake game. All CSS and JavaScript must be inline in one .html file — no external libraries, no imports, no dependencies. The game should have a dark background, render on an HTML canvas, use arrow keys to control the snake, track and display the score, and show a game-over screen with the final score when the snake hits a wall or itself. Start the game automatically on page load. The visual style should feel retro and minimal. This is a one-shot challenge, so you get one chance to get it right. There will be no followup prompt. I will post the generated html file to GitHub for scoring and comparison against the output from other LLMs. I will submit the game to https://discord.gg/dcSyD75Mas for community scoring.

Most LLMs provide a download link for the generated file — click it and save directly to your machine. If no download link is provided, copy the HTML from the response and paste it into a plain-text editor, then save it manually. Either way, use a descriptive, lowercase, hyphen-separated filename — for example snake.html. Avoid spaces; they cause URL encoding headaches.

  1. Save or download the .html file to your machine.
  2. Double-click the file to open it in a browser.
  3. Verify it loads and runs.
✦
Works Locally = Works on GitHub Pages If the game runs in your browser from a local file, it will run identically on GitHub Pages. No server-side magic involved. If it's broken — that's the game you're shipping. One shot.
STEP 05 Upload Your Game to GitHub

There are two ways to get your file into the repo. Pick the one that matches your comfort level:

Option A — Browser Upload (No Tools Required)

  1. Go to your repo on GitHub.
  2. Click Add file → Upload files.
  3. Drag your .html file into the upload box, or click to browse for it.
  4. Scroll down to the Commit changes section. Add a short message like Add lunar-lander game.
  5. Make sure Commit directly to the main branch is selected.
  6. Click Commit changes.

Option B — Git Command Line (Faster for Repeat Uploads)

If you have Git installed, clone the repo once and then push files without touching the browser:

TERMINAL# Clone the repo to your machine (do this once) git clone https://github.com/YOUR-USERNAME/YOUR-REPO.git cd YOUR-REPO # Copy your game file into the repo folder, then: git add my-game.html git commit -m "Add my-game one-shot" git push origin main
ℹ
Installing Git Download Git from https://git-scm.com. On Windows, use Git Bash. On Mac, the Xcode Command Line Tools include Git (run git --version in Terminal — it will prompt an install if missing).
STEP 06 Verify the Game Is Live

After committing your file, GitHub Pages rebuilds the site automatically. To check the deployment status:

  1. In your repo, click the Actions tab.
  2. You'll see a workflow called pages build and deployment running. A yellow dot means it's still building. A green checkmark means it's live.
  3. Once green, visit your game's URL:
GAME URL PATTERNhttps://YOUR-USERNAME.github.io/my-game.html # Example: https://thedailybabel.github.io/lunar-lander.html

Open that URL in any browser. If the game loads — congratulations, you've shipped. Share the link. It works on mobile, tablet, and desktop without any installation.

✦
Custom Domain GitHub Pages supports custom domains. If you own a domain, go to Settings → Pages → Custom domain, enter your domain, and update your DNS with a CNAME record pointing to YOUR-USERNAME.github.io. HTTPS is provisioned automatically via Let's Encrypt.
STEP 07 Build a Game Index Page (Optional but Recommended)

Once you have more than one game, create an index.html at the root of your repo. This file is what GitHub Pages serves when someone visits your root URL — it becomes your game launcher, your arcade cabinet menu.

A simple index links to each game file by name. When visitors land on your site they see a list of available games rather than a GitHub 404 or directory listing. Add a new link every time you add a game:

INDEX SNIPPET<!-- index.html (root of your repo) --> <a href="lunar-lander.html">🚀 Lunar Lander</a> <a href="lucky-sevens-slots.html">🎰 Lucky Sevens Slots</a> <a href="castle-wolfenstein.html">🏰 Castle Wolfenstein</a>

Style it however you like. The file still follows the same one-shot rule: pure HTML + CSS + JS, all inline, committed directly to the repo.
